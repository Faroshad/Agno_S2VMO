SECTION 1 — STRAIN GAUGE SENSOR DATA
────────────────────────────────────────────────────────────────────────────────

Source file: sensors/realistic_sensor_generator.py
              tools/sensor_update.py

Number of gauges : 4 (fixed, hard-coded)
Sensor IDs       : S1, S2, S3, S4
Physical unit    : microstrain (μE)
Value range      : clipped to [-1500, 1500] μE
Precision        : rounded to 2 decimal places (float)
Update cadence   : ~0.33 Hz (default 3.0 s period via SimulationCoordinator)
                   configurable via --period-s CLI argument

Sensor grid positions (fixed):
  Sensor  grid_i  grid_j  grid_k
  ------  ------  ------  ------
  S1        20      15      40
  S2        50      15      40
  S3        35      25      30
  S4        65      10      50

Per-reading dictionary structure (returned by generate_strain_readings(t)):
  Key                Type     Description
  -----------------  -------  ------------------------------------------------
  strain_uE          float    Strain in microstrain (core measurement)
  grid_i             int      Voxel grid index i (x-axis)
  grid_j             int      Voxel grid index j (y-axis)
  grid_k             int      Voxel grid index k (z-axis)
  timestamp_ms       float    Wall-clock timestamp in milliseconds
  sequence           int      Monotonic sequence counter
  scenario_time_s    float    Scaled elapsed scenario time (seconds)
  env_factor_uE      float    Environmental noise/offset component (μE)

FEM pipeline input vector:
  get_fem_input(t) → list[float, float, float, float]
  = [S1.strain_uE, S2.strain_uE, S3.strain_uE, S4.strain_uE]

Incremental JSON path (UpdateAgent / IncrementalGraphUpdater):
  Strain also appears as the field  strain_uE  (float, μE) on Voxel nodes,
  alongside  temp_c (temperature °C),  load_N (load in Newtons),
  hx711_raw (raw ADC integer from HX711 load cell chip).

SECTION 2 — GYROSCOPE (IMU) SENSOR DATA
────────────────────────────────────────────────────────────────────────────────

Source file: core/change_detector.py
              tools/neo4j_tools.py  (DATABASE_SCHEMA)

NOTE: Gyroscope data is NOT part of the default simulation pipeline (no
RealisticSensorGenerator path for gyro). It is only present in the incremental
JSON → Neo4j update path (UpdateAgent). If your external hardware/JSON feed
includes gyro data it will be ingested; otherwise these fields remain absent.

Physical unit    : degrees per second (°/s)
Axes             : X, Y, Z (3-axis)

Input JSON structure (nested under each voxel record):
  {
    "gyro_dps": {
      "x": <float>,   // rotation rate around X axis, °/s
      "y": <float>,   // rotation rate around Y axis, °/s
      "z": <float>    // rotation rate around Z axis, °/s
    }
  }

Neo4j Voxel property names (flattened on ingest):
  gyro_dps_x   float   °/s
  gyro_dps_y   float   °/s
  gyro_dps_z   float   °/s

Companion IMU data (accelerometer, same pattern):
  Input JSON key : "acc_g"  →  { "x": …, "y": …, "z": … }
  Neo4j properties:
    acc_g_x    float   g (acceleration in units of g)
    acc_g_y    float   g
    acc_g_z    float   g

Sample rate: Not defined in code — whatever rate your external JSON file
             provides.  No Hz constant is set for gyro/acc in this repo.

SECTION 5 — PRE-TRAINED MODELS: ARCHITECTURE & I/O
────────────────────────────────────────────────────────────────────────────────

Source: tools/fem_tool.py  +  models/ directory

Model files:

━━━ MODEL 1: SensorToVoxelNet (S2V) ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Purpose : Map 4 discrete sensor readings to a full 3-D strain field

Input:
  Shape     : (1, 4)  float32 tensor
  Values    : [S1, S2, S3, S4] in dimensionless strain (μE × 1e-6)
  Example   : [500 μE, -200 μE, 300 μE, 800 μE]
                → [5.0e-4, -2.0e-4, 3.0e-4, 8.0e-4]

Output:
  Name      : eps_voxel
  Shape     : (3, 83, 45, 87)  float32
  Channels  : 0=ε_xx, 1=ε_yy, 2=ε_zz   (normal strain components)
  Grid size : 83 × 45 × 87 voxels  (FIXED — must match trained weights)
  Units     : dimensionless strain

━━━ NORMALIZATION (between Stage 1 and Stage 2) ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  strain_mean = np.load('models/strain_mean.npy')   shape matches eps_voxel
  strain_std  = np.load('models/strain_std.npy')    shape matches eps_voxel

  eps_norm = (eps_voxel - strain_mean) / (strain_std + 1e-8)
  → Shape: (3, 83, 45, 87)  — input to StressUNet

━━━ MODEL 2: StressUNet ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Purpose : Predict full 6-component stress tensor field from normalized strain

Input:
  Name      : eps_norm
  Shape     : (3, 83, 45, 87)  float32 (normalized strain, 3 channels)

Output:
  Name      : sigma_voxel
  Shape     : (6, 83, 45, 87)  float32
  Channels  : 0=σ_xx, 1=σ_yy, 2=σ_zz, 3=σ_xy, 4=σ_yz, 5=σ_xz
  Units     : Pascals (Pa)

Derived scalar (post-processing):
  stress_magnitude = √(σ_xx² + σ_yy² + σ_zz²)   per-voxel scalar (Pa)

━━━ PIPELINE SUMMARY TABLE ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  Stage         Input                      Output
  ------------  -------------------------  -----------------------------------
  Raw sensor    4 × μE (float list)        4 × dimensionless (×1e-6)
  S2V model     (1,4) float32 tensor       (3,83,45,87) eps_voxel
  Normalize     eps_voxel + mean/std .npy  (3,83,45,87) eps_norm
  UNet model    (3,83,45,87) eps_norm      (6,83,45,87) sigma_voxel
  Post-process  sigma_voxel channels 0-2   (83,45,87) stress_magnitude

SECTION 7 — VOXEL GRID GEOMETRY
────────────────────────────────────────────────────────────────────────────────

  Grid dimensions  : 83 × 45 × 87  (fixed, must match model weights)
  Voxel size       : pitch (default 0.1 m = 10 cm per voxel)
  Origin           : origin[3] (x,y,z of voxel 0,0,0 in world space, meters)
  World position   : P(i,j,k) = origin + [i, j, k] * pitch

  Sensor locations in the grid:
    S1 at (20, 15, 40)  → world ≈ origin + (2.0, 1.5, 4.0) m
    S2 at (50, 15, 40)  → world ≈ origin + (5.0, 1.5, 4.0) m
    S3 at (35, 25, 30)  → world ≈ origin + (3.5, 2.5, 3.0) m
    S4 at (65, 10, 50)  → world ≈ origin + (6.5, 1.0, 5.0) m

  Solid voxels  : matrix[i,j,k] == True  (inside structural body)
  Empty voxels  : matrix[i,j,k] == False (air / void — excluded from FEM)

SECTION 10 — KEY CONCEPTS & DESIGN NOTES

1. THREE DISTINCT "STRAIN" CONCEPTS in this codebase:
   a) sensor_strain_uE  — raw gauge reading on 4 specific Voxel nodes (μE)
   b) strain_uE         — JSON incremental path field (μE, any Voxel)
   c) eps_xx/yy/zz      — FEM-computed full-field strain arrays on all solid
                          Voxel nodes (dimensionless)

2. GYROSCOPE is SCHEMA-ONLY in the default sim pipeline.
   It exists as Neo4j Voxel properties (gyro_dps_x/y/z) only when external
   JSON data provides it via UpdateAgent. The RealisticSensorGenerator and
   the FEM pipeline do NOT generate or use gyroscope readings.

3. NEURAL NETWORK GRID SIZE (83 × 45 × 87) IS FIXED.
   The model weights were trained for this exact grid. Changing the mesh or
   voxel pitch will break the pipeline unless models are retrained.

4. STRESS UNITS: All stress values (sigma_*, stress_magnitude) are in Pascals.
   Thresholds are also in Pascals (defaults: warning=2000 Pa, critical=3000 Pa).

5. STRAIN SENSOR POSITIONS ARE HARD-CODED at 4 specific grid indices.
   They are replicated in three files: realistic_sensor_generator.py,
   sensor_update.py, and voxel_grid_to_neo4j.py. Change all three if needed.

6. FEM HISTORY ARRAYS: Every FEM cycle appends values to lists on Voxel nodes.
   Over time eps_xx, sigma_xx, etc. grow as Neo4j arrays. Use fem_versions and
   fem_timestamps arrays to correlate values to specific analysis runs.

7. VECTOR SEARCH INDEX uses Voxel nodes with integer 'id' property. The mesh
   initialization path creates Voxels with grid_i/j/k but may not set 'id'.
   The JSON incremental path does set 'id'. This creates a potential mismatch
   that may affect vector search on mesh-initialized graphs.
